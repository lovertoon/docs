## Overview

Users apply to become creators. Admins review and approve/reject these applications.

## Accessing Creator Applications

Navigate to **จัดการผู้ใช้ > ครีเอเตอร์ (Users > Creators)**.

## Pending Applications

The sidebar shows a badge with pending application count.

### Application List
View pending applications:
- Applicant name
- Username requested
- Display name
- Applied date
- Status

## Reviewing an Application

<Steps>
  <Step title="Select Application">
    Click on a pending application.
  </Step>
  <Step title="Review Details">
    Examine all submitted information.
  </Step>
  <Step title="Check Portfolio">
    Visit provided portfolio links.
  </Step>
  <Step title="Make Decision">
    Approve or decline with reason.
  </Step>
</Steps>

## Application Information

Review the following:

| Field | What to Check |
|-------|---------------|
| **Username** | Unique, appropriate, follows guidelines |
| **Display Name** | Professional, not misleading |
| **Bio** | Clear description of creator |
| **Reason** | Genuine motivation |
| **Sample Work** | Quality and originality |
| **Portfolio Links** | External work verification |

## Approval Criteria

### Approve If:
- Original work demonstrated
- Professional presentation
- Clear creative vision
- Following platform guidelines
- Genuine application

### Decline If:
- Copyright concerns
- Low quality or effort
- Inappropriate content
- Fake or spam applications
- Policy violations

## Approving an Application

<Steps>
  <Step title="Click Approve">
    Select the approve action.
  </Step>
  <Step title="Confirm">
    Verify your decision.
  </Step>
  <Step title="Notification Sent">
    Applicant receives approval notification.
  </Step>
</Steps>

After approval:
- User role changes to creator
- Creator can access dashboard
- Username is reserved

## Declining an Application

<Steps>
  <Step title="Click Decline">
    Select the decline action.
  </Step>
  <Step title="Provide Reason">
    Enter a clear decline reason.
  </Step>
  <Step title="Submit">
    Applicant receives notification with reason.
  </Step>
</Steps>

### Good Decline Reasons
- "Portfolio shows insufficient original work"
- "Application incomplete, please reapply with sample work"
- "Username not available or inappropriate"
- "Content appears to be from another creator"

### Avoid
- Vague reasons
- Personal comments
- No explanation

## Managing Existing Creators

### View All Creators
See all approved creators:
- Creator information
- Comics published
- Account status
- Activity metrics

### Creator Actions
- View creator profile
- View their content
- Modify role if needed
- Handle issues

## Creator Statistics

Platform creator metrics:
- Total active creators
- Pending applications
- Approval rate
- Creator activity

## Best Practices

<AccordionGroup>
  <Accordion title="Consistent Standards">
    Apply the same criteria to all applicants.
  </Accordion>

  <Accordion title="Timely Review">
    Process applications within 3-5 business days.
  </Accordion>

  <Accordion title="Constructive Feedback">
    Declined applicants should understand why.
  </Accordion>

  <Accordion title="Verify Work">
    Actually check portfolio links when provided.
  </Accordion>
</AccordionGroup>

## Edge Cases

<AccordionGroup>
  <Accordion title="Well-known Creator">
    Verify identity if claiming to be someone notable.
  </Accordion>

  <Accordion title="Young Applicant">
    Ensure they meet any age requirements.
  </Accordion>

  <Accordion title="Reapplication">
    Check if previously declined and whether issues were addressed.
  </Accordion>
</AccordionGroup>
