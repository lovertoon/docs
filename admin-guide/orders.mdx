## Overview

Orders represent chapter purchases made by users. Admins can view and analyze purchase data.

## Accessing Orders

Navigate to **การเงิน > ประวัติการซื้อ (Finance > Orders)**.

## Orders List

View all purchases:

| Column | Information |
|--------|-------------|
| **Order ID** | Unique identifier |
| **User** | Buyer information |
| **Comic** | Purchased content |
| **Chapter** | Specific chapter |
| **Amount** | Coins spent |
| **Date** | Purchase timestamp |

## Filtering Orders

### By Date
- Today
- This week
- This month
- Custom range

### By Comic
Filter orders for specific comic.

### By Creator
View orders for a creator's content.

### By User
Find purchases by specific user.

## Order Details

Click on an order to see:
- Full user information
- Complete comic/chapter details
- Transaction timestamp
- Revenue split breakdown

## Order Analytics

### Revenue Metrics
- Total orders
- Total coins spent
- Revenue by period
- Average order value

### Popular Content
- Most purchased comics
- Top-selling chapters
- Trending purchases

### User Insights
- Top purchasers
- Purchase patterns
- Repeat buyers

## Use Cases

### Support Inquiries
When users report purchase issues:
1. Search for the order
2. Verify transaction status
3. Check user balance
4. Resolve or escalate

### Creator Disputes
When creators question earnings:
1. Filter by creator
2. Show transaction history
3. Verify calculations
4. Explain discrepancies

### Analytics
Understanding platform economics:
- Revenue trends
- Popular pricing
- User behavior
